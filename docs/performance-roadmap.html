Game Store — downloadable games Next up
Games Corner gets a My Games / Store two-tab layout with detail rows (icon, name, description, size, genre tags). Games are not immediately playable — user browses the Store, clicks Install, watches a 1.5 s progress bar, then the game appears in their library. Install state persisted in localStorage per user ID. All 5 non-chess games converted to React.lazy() so no game JS is parsed at startup until installed and launched. Uninstall removes the game from the library.

Pause all animations on focus loss Next up
When the app window loses focus (visibilitychange), a .paused class is toggled on <html>. One CSS rule freezes every @keyframes in the app — orbs, aura rings, typing dots, bubble floats — with zero per-component changes. The Three.js render loop in ChessBoard3D is stopped via gl.setAnimationLoop(null). BubblePop and Tropico hook their existing gamePaused state to a custom aerochat:visibilitychange DOM event so mid-game freezes and resumes cleanly.
~8–12% GPU saved when hidden Games freeze mid-session cleanly App.tsx index.css ChessBoard3D.tsx BubblePop.tsx Tropico.tsx

Throttle Realtime channels when window not focused Shipped
Increase Supabase presence heartbeat interval from 10 s to 30 s when document.hidden. Debounce store updates while hidden. Reduces JS wakeup frequency without losing message delivery.

Memoize Zustand selectors — stop cascade re-renders Shipped
Components selecting large objects (e.g. the full friends array) re-render on every presence tick. Add shallow equality selectors and split subscriptions to primitives to eliminate cascading re-renders during active chats.

Virtualize the message list Shipped
Currently renders all cached messages as DOM nodes (up to 300 per chat). @tanstack/react-virtual renders only visible rows, keeping DOM minimal regardless of history length.

Reduce orb count & compositor layers ✓ Shipped
5+ orb divs across the app each force a separate GPU compositor layer due to filter: blur(). Consolidate to ≤3 total, use will-change: transform only (not filter), consider a shared canvas orb layer.
ChatLayout.tsx ChatWindow.tsx

Scope localStorage message cache keys per user ID ✓ Shipped
Message caches are keyed by contact ID only, not userId:contactId. On a shared device, user2 reads user1's decrypted message cache. Prefix all keys with the authenticated user's ID and prune stale entries on login. (Contact restore key already fixed.)
